### 🔹 2.1. «Плохая» форма — разберите проблемы

**Задача:** реализуйте форму регистрации используя только `useState` и ручную валидацию — намеренно с плохими практиками.

**Форма должна содержать поля:**

* Email
* Пароль и подтверждение пароля
* Роль (select: студент / преподаватель)
* Чекбокс «Принимаю условия»

**Намеренно сделайте следующее (это учебный пример антипаттернов):**

* `useState` — на каждое поле отдельно.
* Валидация email — через `.includes('@')` (не настоящая).
* Проверка совпадения паролей — в `handleSubmit`, вручную.
* При серверной ошибке — вызов `alert()`.
* Кнопка Submit не блокируется во время отправки.
* Нет `aria-invalid` / `aria-describedby`.

**Что нужно сделать:**

1. Добавьте в компонент счётчик ре-рендеров через `useRef` и выведите его на экран.
2. Убедитесь, что он растёт при каждом нажатии клавиши.
3. Напишите в комментарии к коду — почему каждый из этих подходов плохой.

**Контрольная точка:** форма работает, счётчик ре-рендеров растёт, антипаттерны задокументированы в комментариях.

### 🔹 2.2. «Хорошая» форма — React Hook Form + Zod

**Задача:** та же форма регистрации, переписанная правильно.

**Установите зависимости:**

```bash
npm install react-hook-form zod @hookform/resolvers
```

**Шаг 1 — Создайте Zod-схему в отдельном файле `schema.ts`:**

```ts
import { z } from 'zod'

export const schema = z.object({
  firstName: z.string().min(2, 'Минимум 2 символа'),
  lastName:  z.string().min(2, 'Минимум 2 символа'),
  email:     z.string().email('Некорректный email'),
  password:  z.string()
               .min(8, 'Минимум 8 символов')
               .regex(/[A-Z]/, 'Нужна заглавная буква')
               .regex(/[0-9]/, 'Нужна цифра'),
  confirmPassword: z.string(),
  role:      z.string().min(1, 'Выберите роль'),
  agree:     z.boolean().refine(v => v === true, 'Примите условия'),
})
// Кросс-поле: пароли должны совпадать
.refine(
  d => d.password === d.confirmPassword,
  { message: 'Пароли не совпадают', path: ['confirmPassword'] }
)

export type FormData = z.infer<typeof schema>
```

**Шаг 2 — Подключите схему к форме через `zodResolver`:**

```tsx
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { schema, type FormData } from './schema'

const {
  register,
  handleSubmit,
  formState: { errors, isSubmitting },
  setError,
  reset,
} = useForm<FormData>({
  resolver: zodResolver(schema),
  mode: 'onTouched',
})
```

**Шаг 3 — Подключите поля через `register()`:**

```tsx
<input
  id="email"
  type="email"
  aria-invalid={!!errors.email}
  aria-describedby="email-error"
  {...register('email')}
/>
{errors.email && (
  <span id="email-error" role="alert">
    {errors.email.message}
  </span>
)}
```

**Шаг 4 — Обработайте серверную ошибку через `setError()`:**

Сымитируйте ответ сервера: если email содержит `taken@` — вернуть ошибку. Используйте `setError` чтобы ошибка попала прямо в поле, а не в `alert()`:

```tsx
const onSubmit = async (data: FormData) => {
  try {
    await fakeApi(data)
    reset()
  } catch (err) {
    // Серверная ошибка — в поле, не в alert
    setError('email', { message: 'Этот email уже занят' })
  }
}
```

**Шаг 5 — Заблокируйте кнопку при отправке:**

```tsx
<button type="submit" disabled={isSubmitting}>
  {isSubmitting ? 'Отправляем...' : 'Зарегистрироваться'}
</button>
```

**Требования к итоговому компоненту:**

* Счётчик ре-рендеров рядом — покажите, что теперь их почти нет.
* Ошибки появляются inline под каждым полем, не через `alert`.
* При ошибке совпадения паролей — ошибка именно на поле `confirmPassword`.
* Кнопка заблокирована во время отправки.
* Все инпуты имеют `aria-invalid` и `aria-describedby`.

**Контрольная точка:** форма работает, ре-рендеры минимальны, серверная ошибка идёт в поле.

### 🔹 2.3. Альтернативный вариант — Formik + Yup

> Выполните это задание **вместо или в дополнение к** заданию 2.2 — на ваш выбор.

**Установите зависимости:**

```bash
npm install formik yup
```

**Шаг 1 — Создайте Yup-схему:**

```ts
import * as yup from 'yup'

export const schema = yup.object({
  firstName: yup.string().min(2, 'Минимум 2 символа').required('Обязательное поле'),
  email:     yup.string().email('Некорректный email').required('Обязательное поле'),
  password:  yup.string()
               .min(8, 'Минимум 8 символов')
               .matches(/[A-Z]/, 'Нужна заглавная буква')
               .required('Обязательное поле'),
  confirmPassword: yup.string()
               .oneOf([yup.ref('password')], 'Пароли не совпадают')
               .required('Обязательное поле'),
  role:      yup.string().required('Выберите роль'),
  agree:     yup.boolean().oneOf([true], 'Примите условия'),
})
```

**Шаг 2 — Подключите к Formik:**

```tsx
import { useFormik } from 'formik'

const formik = useFormik({
  initialValues: { firstName: '', email: '', password: '',
                   confirmPassword: '', role: '', agree: false },
  validationSchema: schema,
  onSubmit: async (values, { setFieldError, resetForm }) => {
    try {
      await fakeApi(values)
      resetForm()
    } catch {
      setFieldError('email', 'Этот email уже занят')
    }
  },
})
```

**Шаг 3 — Подключите поля:**

```tsx
<input
  id="email"
  name="email"
  value={formik.values.email}
  onChange={formik.handleChange}
  onBlur={formik.handleBlur}
  aria-invalid={!!(formik.touched.email && formik.errors.email)}
/>
{formik.touched.email && formik.errors.email && (
  <span role="alert">{formik.errors.email}</span>
)}
```

**Контрольная точка:** форма работает так же, как в задании 2.2. Сравните подходы — что удобнее и почему?
